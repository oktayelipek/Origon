# Keyboard

On-screen numpad for touch-first amount entry (mobile trade flows).
Source: Figma `- Keyboards` (`12:58173`). Full QWERTY layouts defer to
platform-native keyboards; Origon ships numeric and decimal variants.

## Live preview

<Demo />

export const Demo = () => {
  const [amount, setAmount] = useState('');
  const push = (k) => setAmount((a) => (a + k).slice(0, 12));
  return (
    <div style={{ display: 'flex', flexDirection: 'column', gap: 16, padding: 24, background: 'var(--semantic-level-basement, #0b0f1a)', borderRadius: 12, marginBottom: 24, maxWidth: 320 }}>
      <Input label="Amount" value={amount} onChange={(e) => setAmount(e.target.value)} placeholder="0.00" fullWidth />
      <Keyboard
        layout="decimal"
        onKey={push}
        onBackspace={() => setAmount((a) => a.slice(0, -1))}
        onConfirm={() => alert(`Confirmed: ${amount}`)}
      />
    </div>
  );
};

OrigonKeyboard(
  layout: OrigonKeyboardLayout.decimal,
  onKey: (k) => setState(() => amount += k),
  onBackspace: () => setState(() => amount = amount.substring(0, amount.length - 1)),

## Notes

- Grid: 3×4 with a 5th row for the confirm action.
- Backspace icon is inline SVG (no dependency on icon package).
- No haptics wired up in web build; Flutter/Swift ports will bind to platform haptics.
