# Info Row

Inline status / notification row for surfacing messages within a screen (not a
toast — Toast is a separate ephemeral component). Source: Figma
`FK-InfoRow` (node `12:73137`) — 48 variants across Type, Presence, Tone,
and Boxed axes.

## Accessibility

- Rendered as `role="status" aria-live="polite"` — screen readers announce
  when the row mounts, but don't interrupt.
- Dismiss button carries `aria-label="Dismiss"`.
- Content color contrast is validated against WCAG AA at both `presence`
  levels for all tones.

## Do / don't

- ✅ Use `presence="high"` sparingly — reserve for critical messages that
  demand attention.
- ✅ Use `tone="warning"` only for destructive or risky actions.
- ❌ Don't use InfoRow for validation errors on a specific field — use
  Input's error state instead.
- ❌ Don't stack more than 2 InfoRows in a view — noise.
