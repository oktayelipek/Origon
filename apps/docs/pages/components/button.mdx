# Button

The primary interactive control. Source of truth: Figma component set `{FKButton}`
(node `8:135`) — 106 variants across 8 property axes (Size, Style, Presence,
Type, State, Direction, Icon, Text).

## Anatomy

- **Container** — background/border driven by `variant` + `presence` + state.
- **Icon slot** — optional; positioned before or after the label, or replaces it.
- **Label** — text, mid weight (500), font-size scaled with `size`.
- **Corner radius** — small=28 (pill), medium=8, large=12.

Padding, gap, radius, and text sizes all reference tokens (no hand-tuned numbers).

## States

- **Default** — resting.
- **Hover** — background darkens ~8% (solid) or gets a 6% overlay (outline/ghost).
- **Active** (pressed) — same overlay, deeper (~16% / 10%).
- **Disabled** — `semantic.button.disable` background, `semantic.text.disable`
  text, cursor blocked.

The `focus-visible` ring is provided by the browser default for now — a token
for focus outline will follow when Figma adds one.

## Accessibility

- Rendered as a native `<button>` — inherits keyboard, focus, and screen reader
  behavior for free.
- Icon-only buttons **must** provide `aria-label`.
- `disabled` uses the native attribute (no ARIA needed).

## Do / don't

- ✅ Prefer `variant="primary"` for the primary action on a screen; use it once.
- ✅ Use `variant="outline"` or `"ghost"` for secondary/tertiary actions.
- ✅ Give icon-only buttons a clear `aria-label`.
- ❌ Don't override colors inline — if a design needs a different color, propose
  a new token or variant.
- ❌ Don't put two primary buttons next to each other. Pair `primary` with
  `outline` or `ghost`.
