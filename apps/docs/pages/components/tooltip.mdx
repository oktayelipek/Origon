# Tooltip

Hover/focus popover. Portals to `document.body`, auto-flips when it would
overflow the viewport. Source: Figma `{FK-Tooltip}` (`11:2720`).

## Behavior

- Positioning is `position: fixed` — the tooltip is anchored to the trigger
  via `getBoundingClientRect()` measurements.
- Auto-flips when the preferred side would overflow the viewport, then clamps
  to a 4px viewport margin.
- Portals into `document.body` so the tooltip escapes overflow: hidden
  parents.
- Rendered as `role="tooltip"`.

## Do / don't

- ✅ Use for icon-only buttons to explain their purpose (with `aria-label`
  on the button too).
- ✅ Use to expand acronyms or explain metrics.
- ❌ Don't put critical information in a tooltip — mobile has no hover.
- ❌ Don't nest interactive elements inside a tooltip. Use a Popover for that.
