# Logos

Multi-color brand marks — banks, crypto platforms, payment cards.
Distinct from [Icons](/components/icons):

- **Icons** are single-color, `currentColor`-driven, meant for UI chrome.
- **Logos** preserve their original brand fills. They ship as React
  components too (inline SVG), but you don't control their color.

Source: Figma `- Icons + Logos` page (`12:73747`) — groups Banks & Donation,
Logos, Logo Symbol [Dynamic], Logo Symbol [Square]. **236 logos** exported
via `scripts/export-logos.mjs`.

## Theme-aware `<BrandLogo>`

For banks that ship both Dark and Light variants, use `<BrandLogo name="…" />`.
It watches `document.documentElement[data-theme]` (set by
[`applyTheme()`](/foundations/theming)) and swaps automatically:

<div style={{ display: 'flex', gap: 24, padding: 24, background: 'var(--semantic-level-basement, #0b0f1a)', borderRadius: 12, marginBottom: 24, alignItems: 'center' }}>
  <BrandLogo name="papara" height={28} />
  <BrandLogo name="garanti" height={28} />
  <BrandLogo name="paycell" height={28} />
  <BrandLogo name="ing" height={28} />
</div>

Switch the theme from the navbar to see the logos update in real time.

```tsx
import { BrandLogo } from '@origon/react';

<BrandLogo name="papara" height={28} />
<BrandLogo name="papara" height={28} themeOverride="light" />  // ignore active theme
```

Lookup rules (first match wins):

1. `{name}Dark` / `{name}Light` — canonical themed variants
2. Opposite-theme fallback if the exact theme variant doesn't exist
3. Bare `{name}` — single-variant logos (Bitcoin, Origon marks, etc.)
4. `{name}Logo` — same, with the `Logo` suffix

## Notes

- Colors are baked in. Do not attempt to recolor with `currentColor` — use
  the Icons library for that.
- If you need a specific variant regardless of the active theme, import the
  named component directly (`PaparaDarkLogo`) or pass `themeOverride`.
