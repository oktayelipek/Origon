# Colors

Origon UI colors are split into three layers:

1. **Base** — pure white and black.
2. **Primitive scales** — neutrals and accents on the canonical 50 → 900 steps.
3. **Brand scales** — Kripto and Hisse (more brands to come).

A **semantic layer** (`text.primary`, `surface.background`, `border.focus`, …)
sits on top of these primitives and is defined per product. Semantic tokens
are not yet imported from Figma — track that work in the roadmap.

## Semantic aliases

Semantic tokens map to primitives, and are what product code should use. First
imports from the Button page:

| Alias | Resolves to | Purpose |
| --- | --- | --- |
| `text.reverse` | `blueGray.50` (#070910) | Text on light surfaces |
| `text.secondary` | `coolGray.500` (#858fa6) | Muted / secondary text |
| `text.disable` | `blueGray.400` (#192f43) | Disabled text |
| `button.primary` | `brand.kripto.600` (#005fae) | Default action |
| `button.secondary` | `blueGray.200` (#131f2f) | Secondary action |
| `button.focus` | `coolGray.50` (#f0f4f7) | Focus ring / hover surface |
| `border.level-2` | `blueGray.200` | Subtle border |
| `border.level-4` | `blueGray.400` | Medium border |
| `border.level-5` | `blueGray.500` | Strong border |

Access in code:

```tsx
import { semantic } from '@origon/tokens-react';

<button style={{ background: semantic.button.primary, color: semantic.text.baseWhite }}>
  Buy
</button>
```

> **Note.** These are currently stored as resolved hex values. Once the token
> pipeline supports references, they will point back to their primitive tokens
> so a primitive change flows through automatically.

## Naming rules

- Scales use `<name>.<step>` where step ∈ `{50, 100, …, 900}`. Lower = lighter
  (except `midnightTeal`, where 50 is the lightest usable teal and 900 is
  near-black by design).
- Brand scales live under `brand.<product>` (React) or `brand<Product>`
  (Flutter/Swift due to language conventions).
- CSS variables are kebab-cased: `--color-brand-kripto-600`.

## Do / don't

- ✅ Reference tokens through the semantic layer once it lands
  (`surface.primary` rather than `brand.kripto[600]` directly in product code).
- ✅ Use step 500 as the "brand default" for both scales.
- ❌ Don't hand-pick hex values that "look close". If a color isn't in the
  palette, propose a token addition in Figma first.
